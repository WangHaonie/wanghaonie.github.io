准备工具

配置 Clash for Windows

更多详细教程见 Clash for Windows 官方文档

对于 Clash for Windows，以下简称 cfw。

  1. 点击上方的 cfw 下载链接，在页面下方找到适合你电脑的安装文件，点击就会进行下载。第一个红框的后面写了 ...-win.7z 的是便携版的压缩包文件，下载第二个只有一个 exe 结尾的就是64位的安装文件，上面那个 arm64 是 ARM 架构的 CPU 运行的，下面那个 ia32 就是32位的 CPU。这里，就以64位安装程序为例。

  2. (汉化教程) 如果已经启动 cfw 的话，先把它退出。右键点击右下角系统托盘里的 cfw 图标选择 Quit 就能退出了。然后右键点击桌面上的 cfw 图标，选择 打开文件所在位置，进入 resources 文件夹，将你下载的 app.7z 压缩文件里面的 app.asar 文件拖入这个文件夹里，这时会弹出对话框提示已经有一个名为 app.asar 的文件，选择 替换目标文件 就行，然后再次打开 cfw，可以发现已经汉化了。

  3. 打开软件后，需要先进行设置。

  4. 点击软件界面左侧的 设置 选项卡，将 通知(会在软件启动的时候和配置文件更新失败的时候显示通知) 和 启动时自动最小化到通知栏(让 cfw 启动就后台运行而不是直接显示界面) 打开，下面的 运行时间格式，把它清空(仅是个人爱好 (觉得有点碍眼))。

  5. 往下滑，主题主题选择暗黑，字体设置为 Segoe UI 或其他字体，打开 使用系统表情符号 (Windows 10 之前的系统慎用)

    托盘代理组样式 推荐选择子菜单，防止因为代理组太多导致的右键菜单太高引起的不便。

  6. 往下滑，找到代理，将代理节点条目宽度设置为 200 像素排序规则设置为延迟，延迟测试超过时间设置为 400 毫秒 (防止原本的 3000 毫秒导致的测试延迟时间很长)，然后勾选 隐藏不可选择的代理组，在 连接 这里把 更改代理时中断连接 设置为全部，开启 配置更改时中断连接，代理模式更改时中断连接；开启 显示进程，将 连接面板中显示的链路信息类型 更改为 全部。

  7. 往下滑，找到 日志 里的 预加载设置为 100 行。

  8. 右键点击左侧的 选项卡 选择 重新排序，将 配置 按钮拖动到 代理 的上面，因为本人觉得代理是包含在配置里面的，所以先有配置，再有代理。

  9. 然后点击配置这里输入你购买的订阅地址，点击下载,下载完成后一般会自动切换到这个配置


    右键点击一下载好的配置文件选择设置将更新间隔设置为一小时

  10. 然后点击左侧的 代理 就可以看到节点了，点击右上角的 WiFi 图标，可以测试延迟，等待它自动排序，选择延迟最小的那个。

  11. 然后点击常规打开允许局域网连接。

  12. 然后在 UWP 应用网络回环 (可以使 UWP 应用走代理) 点击 启动助手，在弹出的界面中 点击 Exempt All，然后点击旁边的 Save Changes。

Windows 11 会在这时报错 0x6C6，如下图所示

可以先关闭这个助手，然后重新启动助手，点击 Refresh 然后再点击 Exempt All，然后再点击 Save Changes。

这时有些同学可能就已经正常了设置了，但是有些时候还会有下图的报错 0x57

点击Exempt None，然后点击 Save Changes，然后关闭窗口，再启动助手，点击 Refresh，再点击 Exempt All，再点 Save Changes，就可以正常保存了。

如果还是失败，重试几下，按照 “Refresh - Exempt All/None - Save Changes - 重新打开” 的方式，总会成功的 (如果你平时不用 UWP 应用，那么就可以跳过了)

  1. 回到 cfw 软件点击左侧的 常规，点击 服务模式 后面的 管理，选择 安装，等待软件自动重启。

    然后打开下面的 TUN 模式，系统代理，开机启动。最后鼠标移动到右下角的网络图标，看见一个叫 Clash 的网卡已经显示 Internet 访问，说明配置成功，可以正常的上网了。

搭建 SubConverter

  1. 下载 SubConverter 并解压到任意一个文件夹 (这里是 C:\SubConverter\)

  2. 打开 cfw 的设置，往下滑找到 子进程，点击编辑。

  3. 将这段代码填入，注意 SubConverter 的路径需要根据实际情况做修改，最后记得点击右下角的保存。

  4. 最后再重启一下 cfw。

  5. cfw 重新打开时，如果出现这个对话框说明子进程配置成功 (出现这个对话框代表 SubConverter 是第一次启动的，如果你是用的是之前用过的 SubConverter，那就不会显示这个了)。没有出现的话，看一下任务管理器里是否有 subconverter.exe 这个进程，没有的话，就直接改用手动打开的方式，当要进行订阅转换的时候，手动打开。

  6. 准备一个非 Clash 订阅的订阅地址 (这里以 https://raw.githubusercontent.com/ssrsub/ssr/master/V2Ray 为例)，打开 URL Encoder，进不去的话用 站长之家 UrlEncode 来重新编码一下链接。

    小声BB：好像不用编码也可以诶，快去试试！

  7. 创建一个文本文档 (方便修改和保存)，按照格式输入以下内容：
    格式：http://127.0.0.1:25500/sub?target=clash&url=<订阅地址>&filename=<处理完成后的文件名>&emoji=true&add_emoji=true&remove_emoji=false&udp=true
    如：http://127.0.0.1:25500/sub?target=clash&url=https%3A%2F%2Fraw.githubusercontent.com%2Fssrsub%2Fssr%2Fmaster%2FV2Ray&filename=v2raySub&emoji=true&add_emoji=true&remove_emoji=false&udp=true

  8. 将这段内容填写到 cfw 的 配置 页面进行下载就行。不出意外的话，一会儿过后就有一个由你命名的，地址为 127.0.0.1:25500 的配置文件出现

  9. 节点一切正常，可以使用

  10. SubConverter 还有其他高级功能，动手能力强的可以去学习一下：
    https://github.com/tindy2013/subconverter/blob/master/README-cn.md
