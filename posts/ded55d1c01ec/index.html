本文最后更新于：9 天前

简单科普一下

规则组

  1. 规则组：rules，顾名思义是用来分流的，什么网站对应什么代理组走不走代理都由它决定。

代理组

  1. 代理组：proxy-groups，通常包含了订阅的所有节点。具体含义不必了解，只需要知道它长这样就 OK 了

    规则组和代理组必须两个都要制作，缺一不可，不能只做其中一个

准备工具

  1. SubConverter
    用于强制订阅配置使用定制的代理组
  2. ConnersHua 规则组 (第三方库，官方已删，注意风险)
    用于提供更多的分流规则
  3. ACL4SSR 规则组
    用于提供更多的分流规则
  4. Visual Studio Code (或者其他除记事本以外的文本编辑器)
    用于编辑配置文件

    实际上 SubConverter 自带了以上的所有规则组，但由于软件本身使用的是离线的规则组，所以有些规则组不是最新的，最好就使用在线的规则组。(虽然规则组这东西更新频率本来就不高，而且不是最新的影响也不大)

定制规则组

重要事项

  1. 在开始定制规则组之前，要必须先明白一个重要的思路，就是规则组的配置文件遵循先后顺序，一个规则放置的位置不一样，结果代理的时候识别的顺序就不一样，就会导致想要的网站得不到正确的分流方式。

    主要是 Clash 核心就是这样读取配置文件的。比如说在一个配置文件里，可能因为导入规则时未去重，导致出现了同一个网站对应两个不同的代理组的情况，这时 Clash 只会以第一条为准，若第一条让代理就代理，让直连就直连。这也就是用 Clash for Windows 配置文件预处理 修改 rules 的原理，将需要走特定规则的网站填入第一列，这样 Clash 只会以这第一次出现的为准，剩下在其他位置的都不算。

  2. 那么怎样才能知道哪些规则组先写前面，哪些规则组先写后面呢？很简单，小范围写前面，大范围写后面就对了。什么意思呢？举个简单的例子，比如说假设现在有两个规则组一个是包含了中国所有网站的规则组 (ChinaAll.list)，另一个是只有哔哩哔哩网站的规则组 (Bilibili.list)，很显然，前者的范围肯定就比后者大，因为哔哩哔哩的网站也在中国所有的网站的范围之内 (特殊情况除外)。
  3. 现在我想要这两个规则组同时出现在一个配置文件里面，而且要保证我能对哔哩哔哩网站进行灵活的分流。当我想看港澳台地区的番剧的时候，我要让它走代理，平时看的话就走直连，那么这两个规则文件应该怎么放呢？那肯定是要把哔哩哔哩网站的规则放在前面，把中国所有网站的规则组放在后面。因为只有你把哔哩哔哩的规则组放在前面了，Clash 才会第一时间识别到这是哔哩哔哩的网站，后期就可以直接专门给哔哩哔哩的网站设置一个代理组来控制是否代理和直连。如果你把它放在中国所有网站的后面了，那这个优先级就变了。就会导致最后哔哩哔哩也走了直连，似乎不影响，但你会发现你无法专门的将它设置成代理，除非你就开全局，但是这样会导致其他国内的网站，比如说百度什么的也一起走代理，就会消耗不少的流量。
  4. 应该都看懂了吧，我的思路讲的可能不太清晰，没看懂的话多看几遍。
