本文最后更新于：2 个月前

提示：UUP dump 只能下载 Windows 10 和 Windows 11 的镜像，其他版本的 Windows 镜像还是推荐在 MSDN 下载

工作原理

我们宣布将我们的下一代交付技术整合到最新的内部构建中，称为统一更新平台 (UUP)。

UUP 最大的社区和用户好处之一是你可以在 PC 上看到下载大小的缩小。我们在我们的构建和发布系统中融合了技术，使所有构建在移动和 PC 操作系统上的设备都可以进行不同的下载。差异下载包只包含自上次更新设备以来所做的更改，而不是完整的构建。当我们推出 UUP 时，这将最终对 PC 产生影响，用户可以期望他们的下载大小减少约 35%，当从一个主要的 Windows 更新到另一个。我们现在正致力于此，目标是在 Windows 10 创作者更新后支持这一功能更新; 内部人士很快就会看到这一点。

我们还改进了设备检查更新的方式，使它们更高效。当我们转向 UUP 时，我们减少了发送到客户端设备的更新数据，同时也减少了我们在设备上所做的处理，这对于建立在移动操作系统上的设备尤其重要。使用 UUP，当你的设备检查更新，Windows 更新服务将评估哪些更新是由给定的设备需要。然后，Windows Update 服务将这些更新返回到设备以供下载和安装。由于服务正在进行更多的处理，这将导致更新操作的检查更快。需要注意的是，使用 UUP，表面上看起来或行为都不会有什么不同，UUP 是发生在幕后的底层平台和服务优化。—— Windows Blogs

从 Windows 10 (其实是2017年，上面的文章是摘自 2016 年 11 月 3 日的 Windows 博客) 开始，微软就推出了一种很特殊的更新下载方式，叫做 UUP (Unified Update Platform, 统一更新平台)，在这之前，如果你想更新系统的话，就要下载整个系统镜像来进行更新。现在使用 UUP 的话，就可以通过判断当前系统版本来确定升级到最新系统版本需要哪些组件，只需要下载这些组件即可，这就大大减少了下载更新所消耗的时间和安装速度。所以 UUP 本质上就是一堆零散的系统文件，而这个网站就是下这些系统文件，然后把它封装成一个完整的系统镜像。

网站使用

确定要下载的版本

  1. 如果你会科学上网，可以到 维基百科 搜索相关页面

    维基百科：Windows 10
    维基百科：Windows 11

  2. 在右侧栏目就可以看到 Windows 的最新版、Dev 频道等版本号。


  3. 有的同学可能就会说，百度百科里面不是有 Windows 10/11 的页面吗？的确有，但我看了看，都不是最新版本，比维基百科差远了。另外在百度搜索 Windows 最新版本 结果没有一个有用。但是在微软官网也有一个类似的记录 Windows 历史版本的 页面，虽然貌似不是最新的，但也比百度百科的新。
  4. 其实最新版本在 UUP Dump 网站里的 新增内部版本 模块就有，但是眼睛容易看花，也容易看错，仔细点看也没问题。

开始下载镜像制作包

  1. 获取到最新版本号后，我们就可以直接在 UUP Dump 搜索版本号了

    高级操作：搜索组建号也可以，比如 19045、22H1、1903 等

  2. 查找带有这些关键字的链接：(<版本号>)，x64 (64位系统)，amd64 (64位系统)，Cumulative Update (累积更新)，Feature Update To (功能更新) 等。

有些时候在 UUP 下载页面可能会提示以下信息，这时就要去找找这个版本号的更新有没有对应的其他选项了，比如看看这个版本是否还有一个 Cumulative Update 或 Feature Update To 选项了，一般情况下都有。

尽可能的避免下载那些描述比较短或后面带有序号的 UUP，如下图。(小声BB：因为个人觉得可能会出问题，当然你也可以自己尝试一下万一没出问题了呢？)

  1. 找到你想要的链接点进去后，会让你选择语言，这里保持默认的中文简体就行，然后点击 下一步；版本这里我们只勾选 Windows Pro (专业版)，点击下一步；下载方式这里我们就保持默认的 使用 aria2 下载并转换，转换选项这里我们将下面的四个一起勾上，最后点击创建下载包就行。

至于为什么这里只选择专业版，一是因为我们日常装系统装的都是专业版，网上某些人喜欢用的什么专业工作站版，企业版什么的，其实都是专业版的变体，后期使用特定的版本转换工具创建那个虚拟版本就行。二是因为选择多个版本的话，后期制作镜像时。会对你选择的其他版本重新进行创建，会增加制作镜像的时间

关于 转换选项 的 4 个选项：
(1) 当可用时集成更新
这个一定要勾选上，不然的话，创建的镜像文件的版本与你所选择的版本不同。

(2) 在集成更新后运行清理
因为在制作镜像的过程中会产生大量的临时文件，所以勾上这个选项是理所当然的。

(3) 集成 .NET Framework 3.5
根据我的经验来看，某些常见的软件甚至是游戏都会要求 .NET Framewrok 3.5 (有的同学可能就要说了，诶，不对呀，我怎么没遇到过这种情况呢？那就是可能因为这个软件或者游戏的安装程序检测到你的系统里面没有安装 .NET Framewrok 3.5 之后，自动在线下载或直接运行他的安装包里自带的 .NET Framewrok 3.5 安装程序，悄悄的为你安装上了。这时有的同学就会又说了，诶，那我让他悄悄的给我安装不就行了吗？对，话是这么说，但是有的程序它不会那么智能，它检测到你没有安装后直接就来个报错或者弹出对话框让你安装，这就又浪费了一些时间)，勾上这个选项以后就免得后期又在 启用或关闭 Windows 功能 里面下载 .NET Framewrok 3.5 了。

(4) 使用固实压缩 (ESD)
ESD 是一种使用了 LZMS 的压缩算法的镜像封装方式，优点就是使用这种算法压缩后的文件体积都比正常压缩的小；缺点就是，在压缩过程中需要 CPU 进行大量的运算 (同时也会占用大约 1~2 GB 的运行内存)，会造成电脑卡顿 (除非你的 CPU 是性能很好的那种)，因此也不要在 UUP 程序运行过程中操作电脑。

  1. 创建后的下载包是一个压缩文件，需要全部解压到一个名字不包含空格的文件夹，否则会引起错误。比如： D:\uupwork 就是一个非常好的文件夹。
  2. 然后运行里面的 uup_download_windows.cmd，之后就是全自动的制作过程，在这期间尽量不使用电脑。(若下载文件时失败，重新运行程序即可)
  3. 一般制作手机大概在 1~2 个小时，完成后会显示 “Press 0 to exit“ (意思是按 0 退出，当然也可以直接点叉关闭窗口)。
  4. 制作好的镜像会出现在之前解压的文件夹里，以 ISO 为后缀，可以对这个文件进行转移和重命名 (为了方便后期查看，这里我写一下我的命名规则：Windows.10.Pro.x64.Build.19045.2364.UUP.iso。
