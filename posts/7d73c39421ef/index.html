开机启动的软件真的会影响系统启动速度吗?

引言

  1. 网上经常有很多人说不要让软件开机自启动，否则会拖慢开机速度。那么事实真的如此吗？是，但不完全是，看具体说的是哪个启动流程。

分析

  1. Windows 启动过程大致可以分为下面几个步骤：
    (0) 开机自检：Power-on self test (POST)
    (1) 预引导：UEFI/BIOS 固件
    (2) 引导：bootmgr、winload.exe
    (3) 初始化内核：ntoskrnl.exe
    (4) 初始化 Windows：wininit.exe
    (5) Windows 登录：winlogon.exe
    (6) 用户登录：logonui.exe
    (7) 用户初始化：userinit.exe、explorer.exe
  2. 硬件方面：网上说的开机速度往往指的是按下电源键时到进入系统桌面的速度，也就是上方从 (0) 到 (7) 的所有流程。其中每个流程都会受计算机硬件本身的影响，来决定最终的启动速度。比如说第 (0) 的开机自检，如果计算机中某个硬件无法识别，比如说快要坏掉的机械硬盘等，这样就会拖慢系统启动速度。然后从第 (2) 到第 (5) 不仅要不仅关联到处理器计算速度，还要关联到硬件的驱动，因为系统要在这个时候加载驱动程序。如果处理器速度太慢或者是驱动文件太大，或者是要驱动的硬件本身就已经老化也会导致启动速度变慢。
  3. 软件方面：那么开机启动的软件是在哪个时候启动的呢？答案肯定超乎你的想象，他不是在 Windows 启动的时候启动的，他是在用户登录 (第6点) 过后启动的。在这里又要分两种情况。一是 Windows 计划任务启动，会在用户登录界面，也就是欢迎界面的时候启动，如果要启动的软件数量过多，也就会导致显示登录界面的时间过长。另一种就是注册表 HKCU、HKLM 的 Run/RunOnce 键 (RunOnce 键里的只会启动一次) 启动，是在进入桌面后慢慢启动。

小科普：
1. Run 键指的是
HKCU (当前用户): \HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
HKLM (所有用户): \HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
1. RunOnce 键指的是
HKCU (当前用户): \HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce
HKLM (所有用户): \HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce

总结

  1. 计算机硬件自身的性能和老化程度 (特别是机械硬盘) 决定 Windows 启动速度，也就是会决定 Windows 加载界面会显示多久。
  2. 计划任务要启动的程序数量决定了用户登录时的速度，也就是决定了欢迎界面会显示多久。
  3. 注册表的 HKCU、HKLM 的 Run/RunOnce 键下的项目数量决定了进入桌面时还要多久才能操作电脑。(其实进入了桌面就已经可以操作电脑了，只不过系统会在这时启动要启动的程序，会导致一时的卡顿，所以最好等到所有程序启动完成后再操作电脑，这也就是为什么说开完机后要等一下再动电脑。)
